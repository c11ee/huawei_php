### 1. 先定「边界」再写代码

- 接口清单: 每个接口的路径, 方法 (GET/POST), 入参,出参, 是否要登录
- 鉴权方式: 例如 Sanctum (token), 还是内部固定密钥等. 定了再写登录的接口

### 2. 再动 Laravel 里的 [入口]

- routes/api.php: 先只加部分路由, 确保能访问, 返回 JSON

### 3. 有数据再写「库表 + 模型」

- 迁移：表结构（字段、索引、外键）。
- php artisan migrate 跑通。
- Model：和表对应；复杂查询再考虑 Service/Repository。没有持久化需求时（例如纯转发、纯配置），可以暂时跳过迁移。

### 4. 然后写「控制器 + 校验」

- Controller：一个资源或一个业务域一个类，先实现最小逻辑（查库/拼装数组 return response()->json(...)）。
- Form Request（make:request）：入参校验稳定后再抽，避免一开始就把校验写散在控制器里。

### 5. 最后补「横切」能力

- 认证中间件：auth:sanctum 等包在需要登录的路由分组上。
- 统一异常/错误格式、日志、限流：等有若干接口后再收紧也可以。

一句话：从「路由能通 + 返回 JSON」开始，再迁移/模型，再控制器与校验，最后鉴权与统一规范。
